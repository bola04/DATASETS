         <p>
          Just got word that the news came out last night, Haiku is taking part of
          <a href="https://summerofcode.withgoogle.com/organizations/" rel="noopener noreferrer" target="_blank">
           Google Summer of Code 2018
          </a>
          ! As it's been the case, regularly, Haiku will once again be a mentor organization for GSoC, which helps students get involved with open source software development. This will allow not only students become aware of Haiku but will, hopefully, help Haiku itself get fresh blood into the project.
         </p>
         <p>
          The students application period will run from March 12th to the 27th with the projects announced on April 23rd. That I'm aware of, there's no official Haiku GSoC Projects page yet, but there is an
          <a href="https://www.haiku-os.org/community/gsoc/2018/ideas" rel="noopener noreferrer" target="_blank">
           ideas page
          </a>
          , which students can take inspiration from. Let's see what the application period brings to the project!
         </p>

         <p>
          Pulkomandy has released another of his
          <a href="https://www.haiku-os.org/blog/pulkomandy/2017-11-04_haiku_monthly_activity_report_october_2017/" rel="noopener noreferrer" target="_blank">

            GSoC 2017 Project List!
           </a>
          </h2>
         </div>
         <dl class="article-info muted">
          <dt class="article-info-term">
           Details
          </dt>
          <dd class="createdby" itemprop="author" itemscope="" itemtype="https://schema.org/Person">
           Written by
           <span itemprop="name">
            DaaT
           </span>
          </dd>
          <dd class="published">
           <span aria-hidden="true" class="icon-calendar">
           </span>
           <time datetime="2017-05-05T15:21:47+01:00" itemprop="datePublished">
            Published: 05 May 2017
           </time>
          </dd>
         </dl>
         <p>
          As I'm sure you were all aware, Haiku is once again a selected organization taking part of Google's Summer of Code 2017. As usual Haiku had a list of projects/ideas from which students chose one to apply and now the time has come (well, yesterday afternoon) to reveal them. So here's
          <a href="https://summerofcode.withgoogle.com/organizations/6514892937887744/" rel="alternate noopener noreferrer" target="_blank">
           the list of the 7 selected students
          </a>
          and their projects:
         </p>
         <ul>
          <li>
           Vivek Roy -
           <a href="https://www.haiku-os.org/blog/vivek/2017-05-05_%5Bgsoc_2017%5D_3d_hardware_acceleration_in_haiku/" rel="alternate noopener noreferrer" target="_blank">
            3D Acceleration in Haiku
           </a>
           - Port DRM for the i915 to Haiku
          </li>
          <li>
           AkshayAgarwal007 - Calendar Application - Develop a native calendar application
          </li>
          <li>
           digib0y - Haiku Support for new Harfbuzz library
          </li>
          <li>
           Joseph Calvin Hill - Porting the Swift 3.1 Programming Language to Haiku
          </li>
          <li>
           Anirudh M - Preferences GUI Refactoring - Develop a shared/unified control panel
          </li>
          <li>
           Ayush Agrawal - TCP optimization and fine tuning - Improve the current TCP implementation
          </li>
          <li>
           Hy Che - Add write support for Btrfs
          </li>
         </ul>
         <p>
          Here's to hoping all 7 students have fun and reach the end goal of their projects (and if they enjoy it, stick around after the GSoC is over)!
         </p>

         </div>
         <dl class="article-info muted">
          <dt class="article-info-term">
           Details
          </dt>
          <dd class="createdby" itemprop="author" itemscope="" itemtype="https://schema.org/Person">

         <p>
          It's that time of the month again, Pulkomandy posted his latest
          <a href="https://www.haiku-os.org/blog/pulkomandy/2017-04-31_haiku_monthly_activity_report_april_2017/" rel="alternate noopener noreferrer" target="_blank">

          </a>
          , this time for April and it covers hrevs 51064 to 51139. This report covers work on several sections of the OS, including Network, Media and Filesystem, including fixes to the intel_extreme driver, fixes to getaddrinfo, improvement on the GPT partitioning system and improvements to the launch_daemon.
         </p>

           <a href="/index.php/1291-haiku-december-update-report" itemprop="url">
            Haiku December Update Report.
           </a>
          </h2>
         </div>
         <dl class="article-info muted">
          <dt class="article-info-term">
           Details
          </dt>
          <dd class="createdby" itemprop="author" itemscope="" itemtype="https://schema.org/Person">
           Written by
           <span itemprop="name">
            DaaT
           </span>
          </dd>
          <dd class="published">
           <span aria-hidden="true" class="icon-calendar">
           </span>
           <time datetime="2017-01-08T20:59:38+00:00" itemprop="datePublished">
            Published: 08 January 2017
           </time>
          </dd>
         </dl>
         <p>
          Happy New Year! I'd like to wish everyone a great 2017, I really hope it'll be good, I mean, it can't be worse than 2016... can it? I better not jinx it...
         </p>
         <p>
          To start the new year in a nice way, here's Pulkomandy's
          <a href="https://www.haiku-os.org/blog/pulkomandy/2017-01-06_haiku_monthly_activity_report_122016" target="_blank">
           monthly report
          </a>
          with what happened to our lovely OS during the month of December, covering revisions hrev50718 up to hrev50829. In the report we can see work has been done in the user interface - for example an app to notify users when an update is available, instead of doing this via CLI - area, as well as work related to the kernel and drivers, packages and the launch daemon.
         </p>
         <p>
          One big change which happened last month was the merging of jessicah's and tqh's work on UEFI support, which we already mentioned here, and the work continues on this, with serial debugging now working, and related to this, jessicah has also improved Haiku's GPT support.
         </p>
         <p>
          Great work from everyone involved as always, and I hope everyone had a great time over the holidays.
         </p>

         <p>
          Earlier this week, Adrien (aka Pulkomandy) published his
          <a href="https://www.haiku-os.org/blog/pulkomandy/2016-11-29_haiku_monthly_activity_report_november_2016" target="_blank">
           monthly report
          </a>
          , this time focusing of course on November commits and updates. This summary covers hrev50665 up to hrev50717 and in it several areas of Haiku have been touched, from Networking to E-Mail, with others including Media and User Interface.
         </p>

          Notable mentions include a fix which allows Web+ to open local HTML files again, a fix to a crash caused by playing HTML5 video or audio (important for YouTube for example) and the continued work on EFI support by jessicah. She merged her branch, which will allow other devs to tinker with it and, hopefully, get it up to working condition.
         </p>
         <p>

          </a>
          ? Every little bit helps.
