                 <a href="/threads/netbsd-8-0-release.34482/">
                  NetBSD 8.0 Release
                 </a>
                </h4>
                <div class="block-body">
                 <div class="message message--post">
                  <div class="message-inner">
                   <div class="message-cell message-cell--main">
                    <div class="message-content js-messageContent">
                     <header class="message-attribution">
                      <ul class="listInline listInline--bullet u-muted">
                       <li>
                        <a class="username " data-user-id="4795" data-xf-init="member-tooltip" dir="auto" href="/members/darktrym.4795/">
                         darktrym
                        </a>
                       </li>
                       <li>
                        <time class="u-dt" data-date-string="Jul 31, 2018" data-time="1533069428" data-time-string="22:37" datetime="2018-07-31T22:37:08+0200" dir="auto" title="Jul 31, 2018 um 22:37">
                         Gestern um 22:37
                        </time>
                       </li>
                       <li>
                        <a href="/forums/news-ank%C3%BCndigungen.3/">
                         News &amp; Ankündigungen
                        </a>
                       </li>
                       <li>
                        Antworten: 1
                       </li>
                      </ul>
                     </header>
                     <div class="message-userContent lbContainer js-lbContainer" data-lb-caption-desc="darktrym · Jul 31, 2018 um 22:37" data-lb-id="post-304446">
                      <article class="message-body">
                       <div class="bbWrapper">
                        Mit gut einem Jahr Verspätung erscheint das lang ersehnte NetBSD 8.0 Release.
                        <br/>
                        <br/>
                        <div class="bbCodeBlock bbCodeBlock--expandable bbCodeBlock--quote">
                         <div class="bbCodeBlock-content">
                          <div class="bbCodeBlock-expandContent">
                           <ul>
                            <li>
                             USB stack rework, USB3 support added.
                            </li>
                            <li>
                             In-kernel audio mixer (
                             <a class="link link--external" href="http://netbsd.gw.com/cgi-bin/man-cgi?audio_system+9.i386+NetBSD-8.0" rel="nofollow noopener" target="_blank">
                              audio_system(9)
                             </a>
                             ).
                            </li>
                            <li>
                             Reproducible builds (MKREPRO, see
                             <a class="link link--external" href="http://netbsd.gw.com/cgi-bin/man-cgi?mk.conf+5.i386+NetBSD-8.0" rel="nofollow noopener" target="_blank">
                              mk.conf(5)
                             </a>
                             ).
                            </li>
                            <li>
                             Full userland debug information (MKDEBUG, see
                             <a class="link link--external" href="http://netbsd.gw.com/cgi-bin/man-cgi?mk.conf+5.i386+NetBSD-8.0" rel="nofollow noopener" target="_blank">
                              mk.conf(5)
                             </a>
                             ) available. While most install media do not come with them (for size reasons), the debug and xdebug sets can be downloaded and extracted as needed later. They provide full symbol information for all base system and X binaries and libraries and allow better error reporting and (userland) crash analysis.
                            </li>
                            <li>
                             PaX MPROTECT (W^X) memory protection enforced by default on some architectures with fine-grained memory protection and suitable ELF formats: i386, amd64, evbarm, landisk.
                            </li>
                            <li>
                             PaX ASLR (Address Space Layout Randomization) enabled by default on: i386, amd64, evbarm, landisk, sparc64.
                            </li>
                            <li>
                             Position independent executables by default for userland on: i386, amd64, arm, m68k, mips, sh3, sparc64.
                            </li>
                            <li>
                             A new socket layer
                             <a class="link link--external" href="http://netbsd.gw.com/cgi-bin/man-cgi?can+4.i386+NetBSD-8.0" rel="nofollow noopener" target="_blank">
                              can(4)
                             </a>
                             has been added for communication of devices on a CAN bus.
                            </li>
                            <li>
                             A special pseudo interface
                             <a class="link link--external" href="http://netbsd.gw.com/cgi-bin/man-cgi?ipsecif+4.i386+NetBSD-8.0" rel="nofollow noopener" target="_blank">
                              ipsecif(4)
                             </a>
                             for route-based VPNs has been added.
                            </li>
                            <li>
                             Parts of the network stack have been made MP-safe. The kernel option NET_MPSAFE is required to enable this.
                            </li>
                            <li>
                             Hardening of the network stack in general.
                            </li>
                            <li>
                             Various WAPBL (the NetBSD file system "log" option) stability and performance improvements.
                            </li>
                           </ul>
                           <br/>
                           Specific to i386 and amd64 CPUs:
                           <br/>
                           <ul>
                            <li>
                             Meltdown mitigation: SVS (Separate Virtual Space), enabled by default.
                            </li>
                            <li>
                             SpectreV2 mitigation: retpoline (support in gcc), used by default for kernels. Other hardware mitigations are also available.
                            </li>
                            <li>
                             SpectreV4 mitigations available for Intel and AMD.
                            </li>
                            <li>
                             PopSS workaround: user access to debug registers is turned off by default.
                            </li>
                            <li>
                             Lazy FPU saving disabled on vulnerable Intel CPUs ("eagerfpu").
                            </li>
                            <li>
                             SMAP support.
                            </li>
                            <li>
                             Improvement and hardening of the memory layout: W^X, fewer writable pages, better consistency, better performance.
                            </li>
                            <li>
                             (U)EFI bootloader.
                            </li>
                           </ul>
                           <br/>
                           Many evbarm kernels now use FDT (flat device tree) information (loadable at boot time from an external file) for device configuration, the number of kernels has decreased but the number of boards has vastly increased.
                           <br/>
                           Lots of updates to 3rd party software included:
                           <br/>
                           <ul>
                            <li>
                             GCC 5.5 with support for Address Sanitizer and Undefined Behavior Sanitizer
                            </li>
                            <li>
                             GDB 7.12
                            </li>
                            <li>
                             GNU binutils 2.27
                            </li>
                            <li>
                             Clang/LLVM 3.8.1
                            </li>
                            <li>
                             OpenSSH 7.6
                            </li>
                            <li>
                             OpenSSL 1.0.2k
                            </li>
                            <li>
                             mdocml 1.14.1
                            </li>
                            <li>
                             acpica 20170303
                            </li>
                            <li>
                             ntp 4.2.8p11-o
                            </li>
                            <li>
                             dhcpcd 7.0.6
                            </li>
                            <li>
                             Lua 5.3.4
                            </li>
                           </ul>
                          </div>
                          <div class="bbCodeBlock-expandLink">

                            Zum Vergrößern anklicken....

                       <div class="bbWrapper">
                        Das FreeBSD Release Engineering Team hat planmässig FreeBSD 11.2 freigegeben. Es ist der dritte Subrelease der FreeBSD 11 Serie und bringt als solches sowohl kleinere Verbesserungen als auch Sicherheitsupdates mit.
                        <br/>
                        <br/>
                        Wie seit der letzten Änderung des Supportzyklus gilt auch hier: Jede Unterversion wird nur noch 3 Monate nach Erscheinen der folgenden Unterversion unterstützt [1]. Somit ist ein Update von FreeBSD 11.1 innerhalb von 3 Monaten durchzuführen.
                        <br/>
                        <br/>
                        Installationsimages und VM-Images können ab sofort von den Servern heruntergeladen werden. Binärupdates sind per freebsd-update(8) möglich und der Quellcode steht per SVN und Git zur Verfügung.
                        <br/>
                        <br/>
                        Neben Bugfixes und Treiberaktualisierungen sind die wichtigsten Änderungen gegenüber FreeBSD 11.1:
                        <br/>
                        <ul>
                         <li>
                          Ein "Work Around", um Caches zu leeren, bevor ein Mikrocode-Update auf Intel-CPUs gestartet wird.
                         </li>
                         <li>
                          "Wake On LAN" würde für die Geräte "Ice Lake" und "Cannon Lake" aktiviert.
                         </li>
                         <li>
                          Der "AMD64-Memstick-Installer" wurden aktualisiert, um MBR anstelle von GPT zu verwenden, dies um eine bessere Kompatibilität mit Hardware zu erreichen, die beides verarbeiten kann, aber nicht im BIOS-Modus (Legacy-Modus) mit einem GPT-Partitionsschema bootet. Bitte beachten, dass sich dies dies nicht auf das installierte System, sondern nur auf das Installationsmedium auswirkt.
                         </li>
                         <li>
                          Das "disc1.iso" wurde weiter in der Grösse reduziert, damit es auf eine 700MB CD-ROM passt.
                         </li>
                         <li>
                          libxo(3) wurde auf die Upstream-Version 0.9.0 aktualisiert.
                         </li>
                         <li>
                          Diverse Fehlerbehebungen.
                         </li>
                         <li>
                          Das diskinfo(8)-Utility wurde aktualisiert und enthält zwei neue Flags. Neu zeigt "-s" die Plattenidentität und "-p" den physischen Pfad zur Platte, in einem Speichercontroller.
                         </li>
                         <li>
                          Das umount(8)-Utility wurde aktualisiert und enthält nun ein neues Flag "-N", mit dem ein NFS-gemountetes Dateisystem gewaltsam entfernt werden kann.
                         </li>
                         <li>
                          Das ps(8)-Utility wurde aktualisiert, um anzuzeigen, ob ein Prozess Capsicum(4) unterstützt.
                         </li>
                         <li>
                          Das camcontrol(8)-Utility wurde aktualisiert, um ZAC-Informationen (Zoned-device ATA-Befehlssatz) zu enthalten, wenn der Unterbefehl "identify" verwendet wird.
                         </li>
                         <li>
                          Das ifconfig(8)-Utility wurde aktualisiert und enthält nun eine Zufallsoption, die bei Verwendung mit der "Ether-Option" eine zufällige MAC-Adresse für ein Interface erzeugen kann.
                         </li>
                         <li>
                          Das Utility efibootmgr(8) wurde hinzugefügt, mit dem der EFI-Bootmanager angepasst werden kann.
                         </li>
                         <li>
                          Der UEFI Boot Loader hat unzählige Anpassungen und Erweiterungen spendiert bekommen.
                         </li>
                        </ul>

                        FreeBSD 11.2-RELEASE Announcement
 
    The FreeBSD Release Engineering Team is pleased to announce the
    availability of FreeBSD 11.2-RELEASE. This is the third release of the
    stable/11 branch.
 
    Some of the highlights:
 
      * OpenSSH has been updated to version 7.5p1.

        version 6.0.0.

        added.
 
      * Various miscellaneous kernel, userland application, and library
        updates.
 
      * KDE has been updated to version 4.14.3.

 Availability
 
    FreeBSD 11.2-RELEASE is now available for the amd64, i386, powerpc,
    powerpc64, sparc64, armv6, and aarch64 architectures.
 
    FreeBSD 11.2-RELEASE can be installed from bootable ISO images or over
    the network. Some architectures also support installing from a USB memory
    stick. The required files can be downloaded as described in the section
    below.
 
    SHA512 and SHA256 hashes for the release ISO, memory stick, and SD card
    images are included at the bottom of this message.
 
    PGP-signed checksums for the release images are also available at:
 
      * https://www.FreeBSD.org/releases/11.2R/signatures.html
 
    A PGP-signed version of this announcement is available at:
 
      * https://www.FreeBSD.org/releases/11.2R/announce.asc
 
    The purpose of the images provided as part of the release are as follows:
 
    dvd1
 
            This contains everything necessary to install the base FreeBSD
            operating system, the documentation, debugging distribution sets,
            and a small set of pre-built packages aimed at getting a
            graphical workstation up and running. It also supports booting
            into a "livefs" based rescue mode. This should be all you need if
            you can burn and use DVD-sized media.
 
    disc1
 
            This contains the base FreeBSD operating system. It also supports
            booting into a "livefs" based rescue mode. There are no pre-built
            packages.
 
    bootonly
 
            This supports booting a machine using the CDROM drive but does
            not contain the installation distribution sets for installing
            FreeBSD from the CD itself. You would need to perform a network
            based install (e.g., from an HTTP or FTP server) after booting
            from the CD.
 
    memstick
 
            This can be written to an USB memory stick (flash drive) and used
            to do an install on machines capable of booting off USB drives.
            It also supports booting into a "livefs" based rescue mode. There
            are no pre-built packages.
 
            As one example of how to use the memstick image, assuming the USB
            drive appears as /dev/da0 on your machine something like this
            should work:
 
            # dd if=FreeBSD-11.2-RELEASE-amd64-memstick.img \
              of=/dev/da0 bs=1m conv=sync
 
            Be careful to make sure you get the target (of=) correct.
 
    mini-memstick
 
            This can be written to an USB memory stick (flash drive) and used
            to boot a machine, but does not contain the installation
            distribution sets on the medium itself, similar to the bootonly
            image. It also supports booting into a "livefs" based rescue
            mode. There are no pre-built packages.
 
            As one example of how to use the mini-memstick image, assuming
            the USB drive appears as /dev/da0 on your machine something like
            this should work:
 
            # dd if=FreeBSD-11.2-RELEASE-amd64-mini-memstick.img \
              of=/dev/da0 bs=1m conv=sync
 
            Be careful to make sure you get the target (of=) correct.
 
    FreeBSD/arm SD card images
 
            These can be written to an SD card and used to boot the supported
            arm system. The SD card image contains the full FreeBSD
            installation, and can be installed onto SD cards as small as 2GB.
 
            For convenience for those without console access to the system, a
            freebsd user with a password of freebsd is available by default
            for ssh(1) access. Additionally, the root user password is set to
            root, which it is strongly recommended to change the password for
            both users after gaining access to the system.
 
            To write the FreeBSD/arm image to an SD card, use the dd(1)
            utility, replacing KERNEL with the appropriate kernel
            configuration name for the system.
 
            # dd if=FreeBSD-11.2-RELEASE-arm-armv6-KERNEL.img \
              of=/dev/da0 bs=1m conv=sync
 
            Be careful to make sure you get the target (of=) correct.
