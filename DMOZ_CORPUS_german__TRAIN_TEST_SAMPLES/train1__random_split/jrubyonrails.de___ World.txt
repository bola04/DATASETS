       <div class="post-content">
        <div class="asset-content entry-content">
         <div class="asset-body">
          <p>
           Der lange erwartet Augenblick ist da - JRuby 9.0.0.0 ist erschienen.
 9.0.0.0 ist voll kompatibel zu Ruby 2.2 und benötigt Java 7 oder neuer. Im Gegensatz zu  JRuby 1.7.x, wo man noch zwischen Ruby 1.8 oder 1.9 wählen kann, wird ab dieser Version nur noch eine Ruby Version unterstützt.
          </p>
          <p>
           <a href="https://github.com/jruby/jruby/wiki/Roadmap#jruby-9000-current" target="_blank" title="Major Features von JRuby 9.0.0.0">
            Major Features
           </a>
           von JRuby 9.0.0.0 sind:
          </p>
          <blockquote>
           <ul>
            <li>
             Ruby 2.2 compatibility
            </li>
            <li>
             A new optimizing runtime based on a traditional compiler design
            </li>
            <li>
             New POSIX-friendly IO and Process
            </li>
            <li>
             Fully ported encoding/transcoding logic from MRI
            </li>
            New POSIX-friendly IO and Process
 Fully ported encoding/transcoding logic from MRI
           </ul>
          </blockquote>
          <p>
           Probiert die neue Version aus! Sobald wir die neue Version getestet und neue Erkenntnisse zum Upgrade von Projekten 1.7.x zu 9.0.0.0, Performance oder dem Zusammenspiel mit Gems haben, melden wir uns wieder. Es bleibt spannend!
          </p>
          <p>
           Herunterladen kann man sich die neue Version wie immer auf
           <a href="http://www.jruby.org/download" target="_blank">
            http://www.jruby.org/download
           </a>
           . Oder einfach rvm oder rbenv nutzen.
          </p>

          <p>
           Back in early 2013 we published an in depth article about
           <a href="http://www.jrubyonrails.de/2013/03/continuous-integration-for-jasmine--and-casperjs-tests.html">
            Continuous Integration of Javascript and Coffeescript testing with Jasmine and CasperJS
           </a>
           . Both frameworks evolved since then. Despite its severe improvements
           <a href="http://casperjs.org/">
            CasperJS
           </a>
           still remains at major version 1.0 and therefor has not introduced any significant changes how to handle its integration with TeamCity. At the same time Pivotal Labs however introduced a
           <a href="http://jasmine.github.io/2.0/introduction.html">
            major version 2.0
           </a>
           for
           <a href="https://github.com/pivotal/jasmine">
            Jasmine
           </a>
           in December 2013 -
           <a href="https://github.com/pivotal/jasmine/blob/v2.0.0/release_notes/20.md">
            with some very significant changes
           </a>
           .
          </p>
          <p>
           This article is outlining some of the differences of Jasmine 2.0 to its predecessors when it comes to continuous integration with the TeamCity build server and how to update successfully from Jasmine 1.x to the new and shiny 2.0. For more in depth information on how to setup your continuous integration Javascript and Coffeescript testing make sure you read our previous item on the topic.
          </p>

           <a href="http://jruby.org/2013/10/07/jruby-1-7-5.html" target="_blank">
            JRuby Version 1.7.5
           </a>
           ist erschienen. 243 Bugs oder Features wurden behoben/eingepflegt. Weiterhin ist die Unterstützung der Java invokedynamic ausgeschalten. Wer ausprobieren will, ob sein Projekt mit aktiviertem Feature funktiert, kann es mittels
           <code>
            -Xcompile.invokedynamic=true
           </code>
           aktivieren.
           <br/>
           <br/>
           Nach diesem Release wird die JRuby 1.7.x Serie in einen "Git maintanance branch" verschoben. Und der Start für JRuby 9000 hat begonnen.
          </p>

           Herunterladen kann man sich die neue Version wie immer auf
           <a href="http://www.jruby.org/download" target="_blank">
