         <div class="topicTextNotes">
          <div align="left">
           Diese Webseite informiert sie über die
 Eigenschaften des relationalen Open-Source-
 Datenbankmanagementsystems
          </div>
          <div align="center">
           <img align="bottom" alt="graphic" border="0" height="47" src="NotesImages/Zweig6NotesImage1.jpg" width="203"/>
          </div>
          <div align="left">
           Die "Firebird"-Datenbank kann mit einigen
 überzeugenden Argumenten aufwarten
          </div>
          <ul style="margin-top: 0pt; margin-bottom: 0pt">
           <li style="margin-left: -4mm; margin-right: 0mm; padding-left: 0mm;">
            sie ist
            <b>
             kostenlos
            </b>
            , d.h. u.a. es fallen auch
            <b>
             keine
 Lizenzgebühren
            </b>
            an
            <br/>
            und das wiederum heißt für Sie:
            <ul style="margin-top: 0pt; margin-bottom: 0pt">
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              Sie
 müssen keinem Datenbankhersteller gegenüber Rechenschaft
 ablegen
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              Sie
 müssen sich nicht an irgenwelche undurchsichtigen und/oder
 superkomplizierte Lizenzbedingungen halten und haben damit auch
 keinen Aufwand mit der Abrechnung der Lizenzgebühren
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              Sie
 können Ihr auf Basis dieser Open-Source-Datenbank entwickeltes
 Produkt um die sonst fällig werdenden Lizenzkosten billiger
 verkaufen oder aber wahlweise eine höhere Gewinnspanne
 einstreichen
             </li>
            </ul>
           </li>
           <li style="margin-left: -4mm; margin-right: 0mm; padding-left: 0mm;">
            sie ist
 ein
            <b>
             vollwertiges
            </b>
            relationales Datenbankmanagementsystem
            <ul style="margin-top: 0pt; margin-bottom: 0pt">
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              <a href="Open-Source_Datenbank_Firebird_Vorteile-ACID.html#Open-Source_Datenbank_Firebird_Vorteile-ACID">
               <span>
                ACID
               </span>
              </a>
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              <a href="Open-Source_Datenbank_Firebird_Vorteile-MGA.html#Open-Source_Datenbank_Firebird_Vorteile-MGA">
               <span>
                MGA ("Multi-Generational Architecture", aka "Multi Versioning
 Concurrency")
               </span>
              </a>
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              <a href="Open-Source_Datenbank_Firebird_Vorteile-RI.html#Open-Source_Datenbank_Firebird_Vorteile-RI">
               <span>
                Kaskadierte referentielle Integrität
               </span>
              </a>
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              <a href="Open-Source_Datenbank_Firebird_Vorteile-Stored_Pro.html#Open-Source_Datenbank_Firebird_Vorteile-Stored_Procedures">
               <span>
                Stored Procedures und Trigger
               </span>
              </a>
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              <a href="Open-Source_Datenbank_Firebird_Vorteile-Exceptions.html#Open-Source_Datenbank_Firebird_Vorteile-Exceptions">
               <span>
                Exceptions
               </span>
              </a>
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm; font-size: 11pt; font-weight: bold;">
              <span style="font-size:11pt">
               <b>
                SQL-92
               </b>
               Konformität
              </span>
              (überwiegend konform mit
              <b>
               SQL-2003
              </b>
              )
             </li>
            </ul>
           </li>
           <li style="margin-left: -4mm; margin-right: 0mm; padding-left: 0mm;">
            sie ist
 beim Endkunden relativ
            <b>
             unproblematisch
            </b>
            <ul style="margin-top: 0pt; margin-bottom: 0pt">
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              einsetzbar auf unterschiedlichen Plattformen
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              stabil
 und zuverlässig
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              schnell
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              wartungsarm
             </li>
             <li style="margin-left: 1mm; margin-right: 0mm; padding-left: 0mm;">
              ressourcenschonend
             </li>
            </ul>
           </li>
          </ul>
