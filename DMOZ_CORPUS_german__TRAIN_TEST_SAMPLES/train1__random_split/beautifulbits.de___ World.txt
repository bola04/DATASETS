    throttle = function(func, wait, options) {
                 var context, args, result;
                 var timeout = null;
                 var previous = 0;
                 options || (options = {});
                 var later = function() {
                     previous = new Date;
                     timeout = null;
                     result = func.apply(context, args);
                 };
                 return function() {
                     var now = new Date;
                     if (!previous && options.leading === false) previous = now;
                     var remaining = wait - (now - previous);
                     context = this;
                     args = arguments;
                     if (remaining <= 0) {
                         clearTimeout(timeout);
                         timeout = null;
                         previous = now;
                         result = func.apply(context, args);
                     } else if (!timeout && options.trailing !== false) {
                         timeout = setTimeout(later, remaining);
                     }
                     return result;
                 };
             };

     <h2>
      <i>
       <b>
        iBadge
       </b>
      </i>
      IoT-Device Identitäts-Management
     </h2>
     <p>
      Das Zeitalter des "Internet of Things" (IoT) hat begonnen, und mit sich bringt es eine Flut von Innovationen. Leider liegt es in der Natur der 
                         Sache dass diese IoT Innovationen -- das heißt dem Internet angeschlossene Geräte verschiedenster Formen und Funktionen -- viel Wert auf Innovation
                         legen, und weniger Wert auf konservative Faktoren wie
      <span class="red">
       Datensicherheit
      </span>
      oder
      <span class="red">
       Privatsphäre
      </span>
      .
     </p>
     <p>
      Die Innovation der "
      <a href="http://ikv-tech.com/index.php/product/solution/ibadge">
       iBadge
      </a>
      " liegt jedoch genau dort; in ihrer Sicherheit. 
                         Und zwar kann ein jedes IoT-Gerät welches sich mit einem Server verbindet mit einer 
                         iBadge ausgestattet werden, und dadurch vom Server
      <span class="red">
       authentifiziert
      </span>
      , 
                         gegen Imitation geschützt, und sein Inhalt abhörsicher
      <span class="red">
       verschlüsselt
      </span>
      werden.
     </p>
     <p>
      In dieser Infrastruktur übernimmt
      <span class="red">
       beautifulbits
      </span>
      die Rolle des Cloud-Services, welcher IoT-Geräte mit iBadge Ausstattung authentifiziert, 
                         über eine Web-Anwendung verwaltet, und eine Schnittstelle für Apps anbietet.
     </p>
     <h2>
      Das CipherShed Projekt
     </h2>
     <p>
      Die IT Community war geschockt als die Website der bekanntesten und meistgenutzten
      <span class="red">
       Verschlüsselungssoftware TrueCrypt
      </span>
      am 28. Mai 2014 verkündete die Software würde nicht mehr weiterentwickelt, und sei nicht mehr als sicher zu betrachten.
     </p>
     <img alt="CipherShed" src="images/CS.png" style="height: 200px; float: right;"/>
     <p>
      Da TrueCrypt's Code jedoch öffentlich war (wenn auch unter einer Fragwürdigen Lizenz) dauerte es nicht lange bis sich ein 
                         buntes Team aus Unterstützern zusammenfand, mit dem Ziel TrueCrypt am Leben zu erhalten. Das wegen TrueCrypt's problematischer Lizenz kurzfristig
      <a href="http://ciphershed.org">
       "CipherShed"
      </a>
      benannte Projekt entstand. 
                         Das CipherShed Team plant, nachdem bekannte Sicherheitslücken im 
                         TrueCrypt Code behoben wurden, CipherShed unter einer freien freien Lizenz herauszugeben.
     </p>
     <p>
      Seit dem ersten Tag haben wir die
      <span class="red">
       Infrastruktur
      </span>
      für CipherShed bereitgestellt, einschließlich Website, Email und Maillisten, 
                         Forum, Mumble Voicechat Server und mehr. Niklas Lemcke ist außerdem Mitglied im PMC (
      <span class="red">
       Project Management Comittee
      </span>
      ) und ST 
                         (
      <span class="red">
       Security Team
      </span>
      ) von CipherShed.
     </p>
