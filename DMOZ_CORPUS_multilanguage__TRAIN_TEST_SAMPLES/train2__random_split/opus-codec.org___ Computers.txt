    <h1 id="opus-interactive-audio-codec">
     Opus Interactive Audio Codec
    </h1>
    <h2 id="overview">
     Overview
    </h2>
    <p>
     Opus is a totally open, royalty-free, highly versatile audio codec. Opus is unmatched for interactive
 speech and music transmission over the Internet, but is also intended for storage and streaming
 applications. It is standardized by the Internet Engineering Task Force (IETF) as
     <a href="http://tools.ietf.org/html/rfc6716">
      RFC 6716
     </a>
     which incorporated technology from Skype’s SILK codec and Xiph.Org’s CELT codec.
    </p>
    <h2 id="technology">
     Technology
    </h2>
    <p>
     Opus can handle a wide range of audio applications, including Voice over IP, videoconferencing,
 in-game chat, and even remote live music performances. It can scale from low bitrate narrowband
 speech to very high quality stereo music. Supported features are:
    </p>
    <ul>
     <li>
      Bitrates from 6 kb/s to 510 kb/s
     </li>
     <li>
      Sampling rates from 8 kHz (narrowband) to 48 kHz (fullband)
     </li>
     <li>
      Frame sizes from 2.5 ms to 60 ms
     </li>
     <li>
      Support for both constant bitrate (CBR) and variable bitrate (VBR)
     </li>
     <li>
      Audio bandwidth from narrowband to fullband
     </li>
     <li>
      Support for speech and music
     </li>
     <li>
      Support for mono and stereo
     </li>
     <li>
      Support for up to 255 channels (multistream frames)
     </li>
     <li>
      Dynamically adjustable bitrate, audio bandwidth, and frame size
     </li>
     <li>
      Good loss robustness and packet loss concealment (PLC)
     </li>
     <li>
      Floating point and fixed-point implementation
     </li>
    </ul>
    <p>
     You can read the full specification, including the reference implementation, in
     <a href="http://tools.ietf.org/html/rfc6716">
      RFC 6716
     </a>
     .
 An up-to-date implementation of the Opus standard is also available from the
     <a href="/downloads">
      downloads
     </a>
     page.
    </p>

          libopus 1.3-rc
         </a>
        </h3>
        <span class="post-meta">
         Jun 1, 2018
        </span>
       </header>
       <p>
        This
        <a href="https://archive.mozilla.org/pub/opus/opus-1.3-rc.tar.gz">
         Opus 1.3-rc
        </a>
        release candidate of the upcoming Opus 1.3 includes:
       </p>
       <ul>
        <li>
         Making it possible to use SILK down to bitrates around 5 kb/s
        </li>
        <li>
         Using wideband encoding down to 9 kb/s
        </li>
        <li>
         Improving security (including a new –enable-hardening option)
        </li>
        <li>
         Minor quality improvement on tones
        </li>
        <li>
         Improving Ambisonics support (still experimental)
        </li>
        <li>
         Minor bug fixes
        </li>
       </ul>
       <p>
        Again, we’re providing experimental Windows binaries built with 1.3-rc and
 the upcoming opus-tools and libopusenc.
       </p>
       <p>
        Source code:
        <a href="https://archive.mozilla.org/pub/opus/opus-1.3-rc.tar.gz">
         opus-1.3-rc.tar.gz
        </a>
        <br/>
        Win32 binaries (experimental):
        <a href="https://archive.mozilla.org/pub/opus/win32/opus-tools-test-1.3-rc.zip">
         opus-tools-test-1.3-rc.zip
        </a>
       </p>

         <a class="post-link" href="/release/dev/2017/12/21/libopus-1_3_beta.html">
          libopus 1.3-beta
         </a>
        </h3>
        <span class="post-meta">
         Dec 21, 2017
        </span>
       </header>
       <p>
        This
        <a href="https://archive.mozilla.org/pub/opus/opus-1.3-beta.tar.gz">
         Opus 1.3-beta
        </a>
        beta release
 of the upcoming Opus 1.3 includes:
       </p>
       <ul>
        <li>
         Enabling by default the spec fixes in RFC 8251
        </li>
        <li>
         Improvements to the VAD and speech/music classification using an RNN
        </li>
        <li>
         Improvements to stereo speech coding at low bitrate
        </li>
        <li>
         Added support for ambisonics projection using mapping 3 (disabled by default)
        </li>
        <li>
         Fixes to the CELT PLC
        </li>
       </ul>
       <p>
        Additionally, as a way to test the upcoming update to opus-tools, we’re providing Windows binaries
 built with 1.3-beta. These binaries are based on libopusenc, which means opusenc is finally able
 to make use of the Opus delayed-decision feature to make better speech/music transitions.
       </p>
       <p>
        Source code:
        <a href="https://archive.mozilla.org/pub/opus/opus-1.3-beta.tar.gz">
         opus-1.3-beta.tar.gz
        </a>
        <br/>
        Win32 binaries (experimental):
        <a href="https://archive.mozilla.org/pub/opus/win32/opus-tools-test1.zip">
         opus-tools-test1.zip
        </a>

       <header>
        <h3 class="post-head">
         <a class="post-link" href="/release/dev/2017/08/03/opusfile-0_9.html">
          opusfile 0.9
         </a>
        </h3>
        <span class="post-meta">
         Aug 3, 2017
        </span>
       </header>
       <p>
        The opusfile library provides seeking, decode, and playback
 of Opus streams in the Ogg container (.opus files) including
 over http(s) on posix and windows systems.
       </p>
       <p>
        opusfile depends on libopus and libogg.
 The included opusurl library for http(s) access depends on
       </p>
       <p>
        opusfile and openssl.
       </p>
       <p>
        Changes since the v0.8 release:
       </p>
       <ul>
        <li>
         Fix an invalid free with tag handling.
        </li>
        <li>
         Improve handling of corrupt streams.
        </li>
        <li>
         Improve performance on streams with many chain segments.
        </li>
        <li>
         Improve TLS host validation.
        </li>
        <li>
         Align op_raw_total to work better with op_raw_seek.
        </li>
        <li>
         Documentation and build improvements.
        </li>
       </ul>
       <p>
        This release is backward-compatible with the previous
 release. We recommend all users upgrade.
       </p>
       <p>
        The library is functional, but there are likely issues
 we didn’t find in our own testing. Please give feedback
 in #opus on irc.freenode.net or at opus@xiph.org.
       </p>
       <p>
        Programming documentation is available in tree and
        <a href="https://opus-codec.org/docs/">
         online
        </a>
        .
       </p>
       <p>
        Source code:
        <a href="https://downloads.xiph.org/releases/opus/opusfile-0.9.tar.gz">
         opusfile-0.9.tar.gz
        </a>
        ,
        <a href="https://downloads.xiph.org/releases/opus/opusfile-0.9.zip">
         opusfile-0.9.zip
        </a>
        Windows build:
        <a href="https://archive.mozilla.org/pub/opus/win32/opusfile-0.9-win32.zip">
         opusfile-v0.9-win32.zip
        </a>
       </p>
