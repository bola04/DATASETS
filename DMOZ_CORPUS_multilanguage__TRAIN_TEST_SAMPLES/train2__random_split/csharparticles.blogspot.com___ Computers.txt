       <h3 class="post-title">
        Extending SiteMap Xml file
       </h3>
       <div class="post-body">
        <div>
         <div style="clear:both;">
         </div>
         <p>
          Sometimes you need to store information on per page basis. For example you need keywords or publish date. I think good choice for this can became Web.SiteMap.
         </p>
         <p>
          To implement this just add namespace to the root of the XML file. Something like:
          <span class="text-code">
           &lt;sitemap xmlns="http://schemas.microsoft.com/AspNet/SiteMap-File-1.0"
           <strong>
            xmlns:exs="SiteMap-Extend"
           </strong>
           &gt;
          </span>
          Now you can easely add your attributes without any validation warnings. For example:
          <br/>
          <span class="text-code">
           &lt;sitemapnode url="~/Default.aspx" title="Home" description=""
           <strong>
            exs:keywords="test, something, oops"
           </strong>
           &gt;
          </span>
          From this time this attributes accesible from the code like:
          <span class="text-code">
           SiteMap.CurrentNode["exs:keywords"]
          </span>
         </p>
         <p>
          Enjoy!
         </p>

       <h3 class="post-title">
        Asp.Net 2.0 XML back-end
       </h3>
       <div class="post-body">
        <div>
         <div style="clear:both;">
         </div>
         <p>
          In most cases developers use databases for the site back-end. However for small sites or sites where most content is not needed database you can use XML.
         </p>
         <p>
          In Asp.Net 2.0 you can use
          <a href="http://msdn2.microsoft.com/en-us/library/e8d8587a(en-US,VS.80).aspx">
           XmlDataSource
          </a>
          with many databound controls like
          <a href="”" http:="">
           FormView
          </a>
          or
          <a href="”" http:="">
          </a>
          . This provides easy way to show various data.
         </p>
         <p>
          For example I can describe one of the strategies what I use. This is the
          <a href="http://www.chaliy.com/Sources/Default.aspx">
           Sources
          </a>
          folder of my site. On the start page it have list of all sources. They have title, description, screen url, details url and download url. For details page I need the same list of the data plus long description (article). This means that many of the identical data stored in the different places of the site. Of course I want to have one place for this information. How I was resolving this? I was added sources.xml file in the App_Data directory that has all this data. Now for the start page I use
          <a href="”http://msdn2.microsoft.com/en-us/library/system.web.ui.webcontrols.datalist.aspx“">
           DataList
          </a>
          binded to the
          <a href="http://msdn2.microsoft.com/en-us/library/e8d8587a(en-US,VS.80).aspx">
           XmlDataSource
          </a>
          that retrieve information from the sources.xml. For details page I use
          <a href="”" http:="">
           FormView
          </a>
          with
          <a href="http://msdn2.microsoft.com/en-us/library/e8d8587a(en-US,VS.80).aspx">
           XmlDataSource
          </a>
          that retrieve information from the same xml file, however I use XPath that filters this file to show only appropriate information. Clear?
         </p>
         <p>
          I think this technology is useful for many places. However it has big issue...
          <a href="http://msdn2.microsoft.com/en-us/library/e8d8587a(en-US,VS.80).aspx">
           XmlDataSource
          </a>
          is read-only.

          Some time ago I have decided to try to make something like a "Photoshop" in C#. The first problem has appeared was custom cursors. If you are familiar with "Photoshop" you know that when you change brush size the cursor size also changes. This is solution to make this possible in C# . Note: hare will described only black and white cursors. It is possible that I will upgrade this article or create new for 32-bit cursors.
         </p>
         <p>
          Standard realization of class Cursor practically completely repeats Win API for the cursor except for method
          <strong>
           CreateCursor
          </strong>
          . This method is necessary for drawing custom cursor. So we import it from library user32.dll.
         </p>
         <span class="text-code">
          <pre><span style="color:#000000;">[</span><span style="color:#000000;">DllImport</span><span style="color:#000000;">(</span><span style="color:#800080;">"user32.dll"</span><span style="color:#000000;">)</span><span style="color:#000000;">]</span>

          The method needs handle to the current window, coordinates of the cursor's hot spot, the size and mask of the image.
         </p>
         <p>
          <strong>
           The sizes of the cursor
          </strong>
          :
         </p>
         <p>
          Win API contains method
          <strong>
           GetSystemMetrics
          </strong>
          which returns the possible size of cursor, but it always return 32/32 for standard video devices or 64/64 for hi-resolution devices. In the experimental way I have find out that the sizes of the cursor should be multiple 16 px. So we will not use
          <strong>
           GetSystemMetrics
          </strong>
          . Attempt of use of the non-standard size leads to wrong display of the image. As greater accuracy is necessary for us than 16 px we shall take the size of the cursor with a stock. All that remains will be in visible so this is for us!
         </p>
         <p>

          <strong>
           MessageBeep function
          </strong>
         </p>
         <blockquote>
          The
          <b>
           MessageBeep
          </b>
          function plays a waveform sound. The waveform sound for each sound type is identified by an entry in the registry.
          <h4>
           Parameters
          </h4>
          <strong>
           uType:
          </strong>
          Sound type, as identified by an entry in the registry.
         </blockquote>
         <p>
          <strong>
           Beep function
          </strong>
         </p>
         <blockquote>
          The
          <b>
           Beep
          </b>
          function generates simple tones on the speaker. The function is synchronous; it does not return control to its caller until the sound finishes.
          <h4>
           Parameters
          </h4>
          <strong>
           dwFreq:
          </strong>
          Frequency of the sound, in hertz. This parameter must be in the range 37 through 32,767 (0x25 through 0x7FFF).
          <strong>
           dwDuration:
          </strong>
          Duration of the sound, in milliseconds.
          <h4>
           Return Values:
          </h4>
          If the function succeeds, the return value is true. If the function fails, the return value is false.
