   <div class="main">
    <h1>
     Session Management for Emacs
    </h1>
    <div class="sidebar">
     <p class="sidebar">
      <strong>
       Topic:
      </strong>
      <br/>
      »
      <a href="http://www.gnu.org/software/emacs/" title="Emacs at GNU.org">
       Emacs
      </a>
      <br/>
      »
      <a href="http://www.xemacs.org/" title="XEmacs, fka Lucid Emacs">
       XEmacs
      </a>
     </p>
     <a href="http://sourceforge.net">
      <img alt="SourceForge.net Logo" class="sidebar" height="31" src="http://sourceforge.net/sflogo.php?group_id=46019&amp;type=1" width="88"/>
     </a>
    </div>
    <p>
     When you start Emacs, package Session restores various variables (e.g., input
 histories) from your last session.  It also provides a menu containing recently
 changed/visited files and restores the places (e.g., point) of such a file when
 you revisit it.
    </p>
    <p>
     To restore the variables, this package writes a session file
 (
     <code class="file">
      ~/.session
     </code>
     ) when you exit Emacs.  The file includes
 the values of variables which are automatically updated by Emacs during some
 editing operations:
    </p>
    <ul>
     <li>
      Histories of user input.  For example, strings used in a find/replace command,
 names of files you have visited, etc.
     </li>
     <li>
      Contents of registers, whether they are texts or buffer/file positions.  Buffer
 positions are automatically converted to file positions.
     </li>
     <li>
      List of recently copied/cut text blocks to paste, global markers to jump to,
 and other so-called rings.
     </li>
     <li>
      List of recently changed files with their places and some buffer-local
 variables.
     </li>
    </ul>
    <p>
     To restore the places of a recently changed/visited file when you revisit it,
 this packages stores the places of a buffer in a special variable (the list
 mentioned above) when you kill that buffer (this includes exiting Emacs).
 Places are:
    </p>
    <ul>
     <li>
      Point (the cursor position) and mark (e.g., the opposite position when a region
 is highlighted).  These places will be restored, but a region won't be made
 active.
     </li>
     <li>
      The position of the last change.  A new command (bound to
      <kbd class="kbd">
       C-x
 C-/
      </kbd>
      ) can be used to jump to that position.
     </li>
     <li>
      The boundaries if only a part of a buffer was visible.  Reopening the file will
 again restrict/narrow the editing operations to that region.
     </li>
     <li>
      Values of buffer-local variables.  Useful for variables which control some
 temporary editing behavior, e.g.,
      <code class="lisp">
       overwrite-mode
      </code>
      .
     </li>
    </ul>
    <p>
     As opposed to
     <cite>
      desktop.el
     </cite>
     and
     <a href="related.html">
      other
 packages
     </a>
     , Session does not automatically revisits all files from your last
 session, most of which are not interesting anymore.
    </p>
