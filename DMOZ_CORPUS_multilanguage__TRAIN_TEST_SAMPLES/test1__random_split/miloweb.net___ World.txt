    <p class="topics">
     <a name="aboutserver">
      自宅サーバとは？
     </a>
    </p>
    <p>
     自宅のパソコンをWebサーバとしてWWW(World Wide Web)に公開する事を総称して「自宅サーバ」と呼びます。
     <br/>
     WWWに公開するのですから、全世界からあなたのコンピュータ（サーバ）にアクセス出来るワケです。
     <br/>
     Webサーバはもちろん、メールサーバやFTPサーバなども
     <span class="atten">
      全て無料
     </span>
     で立ち上げが可能です。
 ここで紹介するソフトも
     <span class="atten">
      全てフリーソフト
     </span>
     です。
    </p>
    <span class="atten">
     自宅サーバを立ち上げるメリット
    </span>
    <br/>
    <ul>
     <li>
      自分の技術向上に直結している
     </li>
     <li>
      無料のWebサーバの様に制限が無い
     </li>
     <li>
      基本的に容量無制限
     </li>
     <li>
      楽しいと感じる人はとことん楽しめる
     </li>
    </ul>
    <span class="atten">
     自宅サーバを立ち上げるデメリット
    </span>
    <ul>
     <li>
      何と言ってもセキュリティ面が不安
     </li>
     <li>
      電気代がかかる
     </li>
     <li>
      24時間マシンが稼動しているので、騒音対策が必要な場合も・・・
     </li>
    </ul>

    <p class="topics">
     <a name="jyouken">
      前提条件
     </a>
    </p>
    <p>
     このサイトで紹介する方法に関する前提条件です。
    </p>
    <span class="atten">
     インターネット接続環境に関する事柄
    </span>
    <ul>
     <li>
      プロバイダから
      <span class="atten">
       グローバルIPアドレスが割り振られている
      </span>
      （
      <strong>
       非固定
      </strong>
      グローバルIPアドレスでも可能です）。
     </li>
     <li>
      NAT / IP マスカレード（アドレス変換）機能を持ったルータを使用している。
     </li>
     <li>
      プロバイダからサーバの公開を認められている。
     </li>
     <li>
      常時接続（FTTH や xDSL）である。私は Yahoo!BB 12M です。
     </li>
     <li>
      パソコンが2台以上ある（サーバ機用とクライアント用）
     </li>
    </ul>
    <span class="atten">
     サーバ機に関する事柄
    </span>
    <ul>
     <li>
      OS は、Vine Linux2.5 を使用する。
     </li>
     <li>
      サーバ機はサーバ専用機として使用する。
     </li>
     <li>
      サーバ機に NIC ( Network Interface Card : LANボード)を装備している。
     </li>
    </ul>
    <span class="atten">
     注　意
    </span>
    <p>
     一部プロバイダでは、サーバ公開を制限しているプロバイダもある様です。
     <br/>
     自宅サーバ構築前にご使用のプロバイダに確認されることをお勧めいたします。 （2007/02/21 文面修正）
    </p>

    <p class="topics">
     <a name="whylinux">
      何故 Linux なのか
     </a>
    </p>
    <p>
     このサイトを見てくださった方に今さら「Linux は動作が軽くて、無料で・・・」なんて言ってもしょうがないですよね。
     <br/>
     私が Linux（その中でも Vine Linux）を選定した理由は、以下の通りです。
    </p>
    <ul>
     <li>
      <span class="atten">
       サーバするコンピュータのスペックは低くてＯＫ！
      </span>
     </li>
     <li>
      以前、友人に Vine Linux の本を借りた
     </li>
     <li>
      日本語に強そう（Vine は開発者が日本人）
     </li>
     <li>
      <s>
       個人的にRed Hat Linux はミーハーっぽい気がする・・・
      </s>
      <br/>
      Linux を知れば知るほど他のディストリビューションの魅力に気付きます
     </li>
    </ul>
    <p>
     上記、低スペックでもＯＫ！というのは本当です。
     <br/>
     以下に私のサーバ構築開始時のマシン環境を書いておきます。
    </p>
    <ul>
     <li>
      機種名：Fujitsu FMV-5120D5 （
      <a href="http://www.fmworld.net/biz/fmv/product/former/vdt9602/v5166d5.html" target="_blank">
       FMV-5120D5の詳細スペック
      </a>
      ）
     </li>
     <li>
      ＯＳ：Vine Linux 2.5
     </li>
     <li>
      ＣＰＵ：
      <strong>
       Pentium(TM) 120MHz
      </strong>
     </li>
     <li>
      メモリ：
      <strong>
       32MB
      </strong>
     </li>
     <li>
      HDD：
      <strong>
       6G
      </strong>
      （交換しました）
     </li>
     <li>
      NIC：
      <a href="http://www.corega.co.jp/" target="_blank">
       corega
      </a>
      FEther PCI-TXC （問題なく動作しました）
     </li>
     <li>
      ルータ：
      <a href="http://www.melcoinc.co.jp/" target="_blank">
       MELCO BUFFALO
      </a>
      <a href="http://buffalo.melcoinc.co.jp/products/catalog/item/b/blr-tx4/index.html" target="_blank">
