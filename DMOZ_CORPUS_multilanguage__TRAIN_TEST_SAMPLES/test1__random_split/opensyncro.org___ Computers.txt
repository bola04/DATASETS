      <h1>
       OpenSyncro v2.2 released on 27.02.2008
      </h1>
      <p>
       OpenSyncro v2.2 adds support for processing arbitrarily large SQL query result sets, and brings improvements to using temporary files in iterating integration Pipes, which translates to more scalability options. HTTP components allow setting Content-Type header for requests, and can deal with server responses that specify incorrect or no charset encoding information. Querying customer data by last modification date from a Workspace webshop is now possible. The update includes various bugfixes and resolves a compatibility issue with J2SE 1.6. See
       <a href="changelog.shtml">
        Changelog
       </a>
       page for a list of significant improvements and bugfixes since v2.1.
      </p>
      <p>
       The latest version of OpenSyncro is available at the
       <a href="download.shtml">
        download
       </a>
       page.
       Additionally, the distribution files are hosted at
       <a href="http://sourceforge.net/projects/opensyncro/">
        SourceForge.net
       </a>
       . Also by subscribing to  OpenSyncro's
       <a href="http://freshmeat.net/projects/opensyncro/">
        Freshmeat.net
       </a>
       project you will be notified of new updates to the software.
      </p>
      <br/>
      <h1>
       Overview
      </h1>
      <p>
       OpenSyncro is a lightweight, open source enterprise application integration 
         tool written in Java language. It runs on Apache Tomcat application server 
         and stores its configuration data in MySQL database. OpenSyncro is distributed
 	under the
       <a href="http://www.gnu.org/copyleft/gpl.html">
        GNU General Public License
       </a>
       .
      </p>
      <p>
       OpenSyncro was originally developed for connecting
       <a href="http://workspace.smilehouse.com">
        Smilehouse 
         Workspace
       </a>
       webshop to other enterprise applications, for example ERP, CRM and 
         SCM systems, but it can be used independently for integrations 
         between other applications, as well as for various data conversion tasks.
      </p>
      <br/>
      <h1>
       Benefits of OpenSyncro in application integration
      </h1>
      <h2>
       1) Ready-to-use components for data transfers and data conversions
      </h2>
      <ul>
       <li>
        Source components:
       </li>
       <ul>
        <li>
         Built-in: relational databases (SQL/JDBC), FTP, HTTP(S), Web Services, local and network filesystems
        </li>
        <li>
         Smilehouse Workspace export
         <br/>
        </li>
       </ul>
       <li>
        Data conversion components:
       </li>
       <ul>
        <li>
         CSV to XML
        </li>
        <li>
         ASCII to XML, with regular expressions
        </li>
        <li>
         XML-&gt;Any, with standard XSLT (versions 1.0 and 2.0)
        </li>
        <li>
         Accessing JDBC SQL databases and Web Services during data conversion
        </li>
       </ul>
       <li>
        Destination components:
        <ul>
         <li>
          Built-in: relational databases (SQL/JDBC), FTP, HTTP(S), Web Services, local and network filesystems
         </li>
         <li>
          Smilehouse Workspace import
         </li>
        </ul>
       </li>
      </ul>
      <h2>
       2) Dynamic component model
      </h2>
      <ul>
       <li>
        New components can be developed in Java language
       </li>
       <li>
        New components may be loaded run-time
       </li>
       <li>
        One component can be a part of virtually any number of integration processes
       </li>
       <li>
        Web browser based component configuration, no developer tools to install
       </li>
      </ul>
      <h2>
       3) Logging and error handling
      </h2>
      <ul>
       <li>
        Complete logging of data transfer and conversion events at each component 
           with three log levels available: error, warning and debug
       </li>
       <li>
        The framework allows notifying the source application of
           unrecoverable errors in integration process execution (limited support
           for transactions)
       </li>
      </ul>
      <h2>
