      <h3>
       Features
      </h3>
      <ul>
       <li>
        <strong>
         Simple:
        </strong>
        An easy to use command line interface.
       </li>
       <li>
        <strong>
         Scalable:
        </strong>
        Nested Repositories are submodules done right! Version control
 	collections of repositories.
       </li>
       <li>
        <strong>
         Flexible:
        </strong>
        Hybrid mode for binary files that uses a cloud of server for
 	binaries instead of bloating the source repositories.
       </li>
       <li>
        <strong>
         Accurate:
        </strong>
        Tracking of file operations like creates, deletes, and renames.
       </li>
       <li>
        <strong>
         Safe:
        </strong>
        All file accesses validate checksums for integrity. All file
 	writes include redundancy for error correction.
       </li>
       <li>
        <strong>
         Dependable:
        </strong>
        Highly accurate auto-merge that uses the whole history to
 	resolve conflicts. Most other systems use variations of
        <code>
         diff3
        </code>
        .
       </li>
       <li>
        <strong>
         Discernable:
        </strong>
        Source annotations instantly available.
       </li>
       <li>
        <strong>
         Fast:
        </strong>
        High performance and scales to very large repositories.
       </li>
       <li>
        <strong>
         Free:
        </strong>
        Licensed under the
        <a href="http://www.apache.org/licenses/LICENSE-2.0">
         Apache Version 2 license
        </a>
       </li>
      </ul>
     </div>
     <div class="col-xs-12 col-sm-6">
      <h3>
       Clone a project and push changes
      </h3>
      <pre class="code">
     <code>
 $ bk clone bk://bkbits.net/bkdemo/bk_demo
 $ cd bkdemo
 # edit files using your favorite editor
 $ bk -Ux new
 $ bk commit -y"Comments"
 $ bk push
     </code>
   </pre>
      <h3>
       Create a project and commit
      </h3>
      <pre class="code">
     <code>
 $ bk init new_project
 $ cd new_project
 # edit files using your favorite editor
 $ bk -Ux new
 $ bk commit -y"Initial commit"
       </code>
   </pre>
