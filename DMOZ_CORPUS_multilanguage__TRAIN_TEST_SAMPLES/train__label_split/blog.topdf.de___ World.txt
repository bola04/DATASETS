        <h2>
         <a class="title" href="https://blog.topdf.de/?p=711" rel="bookmark">
          CLIQZ, der bessere Browser
         </a>
        </h2>
        <div class="info">
         <span class="date">
          29. Juni 2017
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=711#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Kürzlich wurde ich auf einen jungen alternativen Browser hingewiesen. Die Versprechen der Präsentation des Programms und Berichte über
          <a href="https://www.netzsieger.de/p/cliqz" target="_blank">
           CLIQZ
          </a>
          klingen gut: „der bessere Firefox“, „der sichere Browser“, „ein Browser, welcher nicht schnüffelt“. Eine kurze Recherche zeigt, dass sich ein Blick lohnen kann!
          <br/>
          Normalerweise nutze ich schon sehr lange Google Chrome. Darin habe ich als zusätzliche Erweiterungen lediglich
          <a href="https://de.wikipedia.org/wiki/Ghostery" target="_blank">
           Ghostery
          </a>
          und
          <a href="https://vimium.github.io/" target="_blank">
           Vimium
          </a>
          installiert und habe so ein für mich gutes Maß an Sicherheit, Komfort, Effizienz und Geschwindigkeit erreicht. Das erste Plugin reduziert Werbung und Tracking auf ein Minimum. Das zweite Plugin ermöglicht mir die Navigation mittels Tastatur, da ich damit durch den Verzicht vom Einsatz der Maus effektiver arbeiten kann.
         </p>
         <p>
          CLIQZ basiert auf Mozilla Firefox. Der Vorsprung, den Google Chrome vor Jahren innen hatte, ist mit der Zeit stark geschmolzen. So gilt dies auch für den darauf basierenden CLIQZ. Alles Wichtige ist vorhanden und die Seiten werden schnell und gut dargestellt.
         </p>
         <p>
          Sehr positiv finde ich die Integration von Ghostery, da ich selbst ein fleißiger Nutzer des Plugins im Google Chrome Browser bin. Damit kann Werbung reduziert werden und gleichzeitig die überall durchgeführte Übermittlung persönlicher Daten auf ein Minimum beschränkt werden. Negativ finde ich die fehlende Option, andere PlugIns von Firefox ebenfalls zu installieren, dass schränkt mich persönlich sehr stark ein. Die vom
          <a href="https://cliqz.com/about" target="_blank">
           Hersteller
          </a>
          ins Feld geführten Sicherheitsgründe sind aber aus diesem Blickwinkel nicht widerlegbar. Die Übersetzung der Oberfläche ins Deutsche ist ebenfalls gelungen und wird viele deutschsprachige Nutzer ansprechen.
         </p>
         <p>
          Die Mehrheit der Anwender wird damit einen Browser erhalten, der sinnvoll eingestellt ist bzw. mit wenigen verständlichen Klicks konfiguriert werden kann. Anwender, die ohne tiefere Computer-Kenntnisse einen sichereren Browser nutzen möchten, werden also gut bedient! Anwender, welche Ihren Arbeitsplatz sehr individuell konfigurieren möchten, sollten unter Windows bei Firefox oder Chrome bleiben.
         </p>
         <p>
          Ob das Geschäftsmodell über die Jahre tragfähig bleibt, wird sicher spannend. Der Start ist vielversprechend!
         </p>

        <h2>
         <a class="title" href="https://blog.topdf.de/?p=694" rel="bookmark">
          NDepend 2017, ein Update
         </a>
        </h2>
        <div class="info">
         <span class="date">
          25. April 2017
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=694#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Vor gut 3 Jahren habe ich
          <a href="https://blog.topdf.de/?p=618">
           hier
          </a>
          eine Bewertung von
          <a href="http://www.ndepend.com/" target="_blank">
           NDepend
          </a>
          veröffentlicht. Nun ist vor kurzem der Hinweis auf eine größere Aktualisierung eingetroffen. Diese neue Version habe ich wieder unter die Lupe genommen und in aktuellen Projekten mit Visual Studio 2015 getestet. Für alle Programmierer, welche dieses Tool nicht kennen:
         </p>
         <blockquote>
          <p>
           NDepend ist eine Software, mit der die Softwarequalität eines .NET-Programms überprüft werden kann. Für die Überprüfung und Auswertung stehen sehr viele Möglichkeiten zur Verfügung. Durch historische Betrachtungsmöglichkeiten kann ebenfalls eine zeitliche Entwicklung der Qualität geprüft und sichergestellt werden.
          </p>
         </blockquote>
         <p>
          Die Installation war vertraut, es ist ein wenig Handarbeit angesagt, die einen Software-Entwickler in keinster Weise vor Fragen stellt. Danach ist das PlugIn in Visual Studio einsatzbereit oder mal kann eine lokale Version nutzen.
          <a href="https://www.jetbrains.com/" target="_blank">
           Jetbrains
          </a>
          <a href="https://www.jetbrains.com/resharper/" target="_blank">
           Resharper
          </a>
          beispielsweise nimmt einem bei der Installation etwas mehr Arbeit ab, wirklich schneller ist man aber damit aber nicht fertig. NDepend legt eigene Projektdateien zu einem Visual Studio Projekt an. Damit ändert sich am eigentlich Projekt nichts und Entwickler ohne NDepend und auch die Versionsverwaltung werden nicht beeinflusst. Das gibt einen großen Pluspunkt von mir!
          <br/>
          <a href="https://blog.topdf.de/wp-content/uploads/2017/05/dash.png">
           <img alt="" class="alignright size-medium wp-image-702" height="182" sizes="(max-width: 300px) 100vw, 300px" src="https://blog.topdf.de/wp-content/uploads/2017/05/dash-300x182.png" srcset="https://blog.topdf.de/wp-content/uploads/2017/05/dash-300x182.png 300w, https://blog.topdf.de/wp-content/uploads/2017/05/dash-768x465.png 768w, https://blog.topdf.de/wp-content/uploads/2017/05/dash-1024x620.png 1024w, https://blog.topdf.de/wp-content/uploads/2017/05/dash.png 1499w" width="300"/>
          </a>
          <br/>
          Im den letzten 3 Jahren haben die Entwickler gute Arbeit geleistet. Es wird nun automatisch per Default eine Historie aufgebaut, für Vergleiche herangezogen und auch bei den Auswertungen entsprechend dargestellt. So hat man quasi immer gleichzeitig die Übersicht über alle gefundenen Probleme, kann aber auch erst mal nur die aktuell hinzugekommenen Probleme ansehen. Hier sind viele Einstellmöglichkeiten vorhanden. Ein großer Pluspunkt ist die unglaubliche Anzahl von sinnvollen Optionen, aber zeitweise auch ein Minuspunkt -&gt; man muss erst einmal den Überblick gewinnen und behalten. Auf einem Rechner mit einem normalen Monitor macht das keinen Spaß. Eine Dualscreen-Konfiguration in 4k war hierfür wesentlich effizienter nutzbar.
         </p>
         <p>
          Ich habe NDepend mit Projekte im WinForm, Asp.Net Webform und Asp.Net MVC Bereich ausprobiert. Hier gab es viele sinnvolle Warnungen und Fehler, die im geprüften Quellcode korrigiert werden sollten. Bei Asp.Net mit WebForms wurden aber auch die Zusammenhänge zwischen Aspx-Formular und Code nicht erkannt, was in sehr vielen Fehlern resultierte, die nicht korrigiert werden können. Hier wird der Hersteller sein Regelwerk noch prüfen.
         </p>
         <p>
          Apropos Regeln: Es ist eine klare Ausrichtung auf den professionellen Entwickler mit einem Sinn für guten wartbaren Code erkennbar. Alle vordefinierten Regeln sind gut erklärt und über eine interne Linq-ähnliche Sprache anpassbar.
         </p>
         <p>
          Wie viele Softwarefirmen haben auch die Macher von NDepend ihr Lizenzmodell auf eine Jahresgebühr gestellt. Das ist nachvollziehbar, da ein Produkt wie NDepend eigentlich permanent weiterentwickelt und aktualisiert werden muss, um mit Visual Studio und neuen Funktionalitäten eingesetzt zu werden. Ob der Preise zu hoch oder tief ist, sollte jeder für sich entscheiden. Wer die Softwarequalität seines Teams im Auge behalten will, wir aber sicher nicht an den Kosten scheitern.
         </p>

        <h2>
         <a class="title" href="https://blog.topdf.de/?p=680" rel="bookmark">
          Sprung in der Identity Spalte beim SQL Server 2012
         </a>
        </h2>
        <div class="info">
         <span class="date">
          23. August 2016
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=680#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Kurz nach der Migration einiger Datenbank von einem SQL Server 2008 auf einen SQL Server 2016 ist aufgefallen, dass es große Sprünge in den Identity-Spalten gab, die nicht durch verworfene Datensätze erklärt werden konnten. Dabei ging es jeweils um Sprünge von ca. 1000 oder sogar 10.000. Die Ursache ist in der
          <a href="https://msdn.microsoft.com/en-us/library/ms186775.aspx">
           MSDN
          </a>
          zu finden:
         </p>
         <blockquote>
          <p>
           The identity property on a column does not guarantee the following: … Consecutive values after server restart or other failures – SQL Server might cache identity values for performance reasons and some of the assigned values can be lost during a database failure or server restart. This can result in gaps in the identity value upon insert.
          </p>
         </blockquote>
         <p>
          Dieser Sprung beträgt bei Spalten mit bigint aber bis zu 10.000, bei Spalten mit int immerhin noch 1.000. Da ist sehr viel und bringt einen in einen Erklärungsbedarf. Basierend auf
          <a href="http://www.codeproject.com/Tips/668042/SQL-Server-2012-Auto-Identity-Column-Value-Jump-Is">
           diesem Beitrag
          </a>
          lässt sich das Problem leicht nachstellen:
         </p>

           </tr>
          </table>
          <p class="theCode" style="display:none;">
           insert into MyTestTable(Name) values ('Mr.Jackson'); 
 
 SELECT Id, Name FROM MyTestTable ORDER BY Id Desc;
          </p>
         </div>
         <p>
          <a href="http://blog.topdf.de/wp-content/uploads/2016/08/sql_t_272.png">
           <img alt="SQL Start Option T272" class="alignright size-full wp-image-685" height="375" sizes="(max-width: 321px) 100vw, 321px" src="http://blog.topdf.de/wp-content/uploads/2016/08/sql_t_272.png" srcset="https://blog.topdf.de/wp-content/uploads/2016/08/sql_t_272.png 321w, https://blog.topdf.de/wp-content/uploads/2016/08/sql_t_272-257x300.png 257w" width="321"/>
          </a>
          <br/>
          Damit sollte das Problem sichtbar sein. Eine Option ist die Nutzung von
          <a href="https://msdn.microsoft.com/en-us/library/ff878091.aspx">
           Sequenzen
          </a>
          mit der
          <strong>
           NOCACHE
          </strong>
          Option, anstatt auf IDENTITY Spalten zu setzen. Wenn diese Option zu aufwendig ist, kann auch die Option -T272 für die komplette Instanz genutzt werden. Damit ist das alte Verhalten (bis SQL 2008) wiederhergestellt.
         </p>
         <p>
          Es bleibt noch anzumerken, dass das TRACE Flag T272 das Caching-Verhalten für die Generierung von Identity-Werten deaktiviert. Negative Auswirkungen auf die Performance beim Erzeugen neuer Datensätze sind somit möglich.
         </p>

        <h2>
         <a class="title" href="https://blog.topdf.de/?p=657" rel="bookmark">
          Daten per SQL INSERT ohne Schlüsselspalte kopieren
         </a>
        </h2>
        <div class="info">
         <span class="date">
          5. August 2015
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=657#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Immer wieder steht man vor der Aufgabe, Datenmengen innerhalb einer Tabelle zu kopieren. Typisierte Datenzugriffsschichten, wie das
          <a href="https://entityframework.codeplex.com/" target="_blank">
           Entity Framework
          </a>
          , bieten zwar viel Komfort, lassen aber zukünftige DDL Updates außen vor. Ein Update des Datenmodells, speziell mit ergänzten oder gelöschten Spalten erzwingt dann automatisch ein Update der Anwendung, wenn die Kopierfunktion vollständige Kopien anlegen soll. Bei oft geänderten Tabellen wir das zu einem Problem. Zudem sind komplexere Kopieroperationen direkt auf dem Server mittels einer StoredProcedure meist wesentlich schneller.
         </p>
         <p>
          Hier bietet sich natürlich die Lösung mit einem
          <a href="https://msdn.microsoft.com/de-de/library/ms174335(v=SQL.120).aspx" target="_blank">
           INSERT SQL
          </a>
          an:
         </p>
         <div class="wp_syntax" style="position:relative;">
          <table>
           <tr>
            <td class="code">
             <pre class="sql" style="font-family:monospace;"><span style="color: #993333; font-weight: bold;">INSERT</span> <span style="color: #993333; font-weight: bold;">INTO</span> MyTable <span style="color: #993333; font-weight: bold;">SELECT</span> mt<span style="color: #66cc66;">.*</span> <span style="color: #993333; font-weight: bold;">FROM</span> MyTable mt;</pre>
            </td>
           </tr>
          </table>
          <p class="theCode" style="display:none;">
           INSERT INTO MyTable SELECT mt.* FROM MyTable mt;
          </p>
         </div>
         <p>
          Das Problem mit diesem Befehl sind aber
          <a href="https://msdn.microsoft.com/de-de/library/ms186775(v=sql.120).aspx" target="_blank">
           IDENTITY
          </a>
          Spalten, die nicht übergeben werden dürfen. Dies kann man natürlich durch Angabe der Spaltennamen umgehen. Damit landet man aber wieder am Anfang des Problems. Die Spaltennamen müssen fest definiert oder zuvor aufwändig ermittelt werden.
         </p>
         <p>
          Eine andere Lösung lässt sich jedoch über
          <a href="https://www.simple-talk.com/sql/t-sql-programming/temporary-tables-in-sql-server/" target="_blank">
           temporäre Tabellen
          </a>
          entwickeln. Man erstellt mit
          <strong>
           SELECT INTO
          </strong>
          eine temporäre Tabelle mit den benötigten Daten. Deren Identity-Spalte kann man problemlos verwerfen und danach funktioniert ein INSERT INTO ohne Problem:
         </p>
         <div class="wp_syntax" style="position:relative;">
          <table>
           <tr>
            <td class="code">
             <pre class="sql" style="font-family:monospace;"><span style="color: #993333; font-weight: bold;">CREATE</span> <span style="color: #993333; font-weight: bold;">PROCEDURE</span> dbo<span style="color: #66cc66;">.</span>CopyTable @SourceId <span style="color: #993333; font-weight: bold;">BIGINT</span>
 <span style="color: #993333; font-weight: bold;">AS</span> 
   <span style="color: #993333; font-weight: bold;">SET</span> NOCOUNT <span style="color: #993333; font-weight: bold;">ON</span>;
  
   <span style="color: #993333; font-weight: bold;">IF</span> OBJECT_ID<span style="color: #66cc66;">(</span><span style="color: #ff0000;">'tempdb..#MyTable'</span><span style="color: #66cc66;">)</span> <span style="color: #993333; font-weight: bold;">IS</span> <span style="color: #993333; font-weight: bold;">NOT</span> <span style="color: #993333; font-weight: bold;">NULL</span> <span style="color: #993333; font-weight: bold;">DROP</span> <span style="color: #993333; font-weight: bold;">TABLE</span> #MyTable;
  
   <span style="color: #993333; font-weight: bold;">SELECT</span> mt<span style="color: #66cc66;">.*</span> <span style="color: #993333; font-weight: bold;">INTO</span> #MyTable <span style="color: #993333; font-weight: bold;">FROM</span> MyTable mt <span style="color: #993333; font-weight: bold;">WHERE</span> mt<span style="color: #66cc66;">.</span>Id <span style="color: #66cc66;">=</span> @SourceId;
  
   <span style="color: #993333; font-weight: bold;">ALTER</span> <span style="color: #993333; font-weight: bold;">TABLE</span> #MyTable <span style="color: #993333; font-weight: bold;">DROP</span> <span style="color: #993333; font-weight: bold;">COLUMN</span> Id;
  
   <span style="color: #993333; font-weight: bold;">INSERT</span> <span style="color: #993333; font-weight: bold;">INTO</span> MyTable <span style="color: #993333; font-weight: bold;">SELECT</span> mt<span style="color: #66cc66;">.*</span> <span style="color: #993333; font-weight: bold;">FROM</span> #MyTable mt;
  
   <span style="color: #993333; font-weight: bold;">IF</span> OBJECT_ID<span style="color: #66cc66;">(</span><span style="color: #ff0000;">'tempdb..#MyTable'</span><span style="color: #66cc66;">)</span> <span style="color: #993333; font-weight: bold;">IS</span> <span style="color: #993333; font-weight: bold;">NOT</span> <span style="color: #993333; font-weight: bold;">NULL</span> <span style="color: #993333; font-weight: bold;">DROP</span> <span style="color: #993333; font-weight: bold;">TABLE</span> #MyTable;
  
   <span style="color: #993333; font-weight: bold;">SET</span> NOCOUNT OFF;
 <span style="color: #993333; font-weight: bold;">GO</span></pre>
            </td>
           </tr>
          </table>
          <p class="theCode" style="display:none;">
           CREATE PROCEDURE dbo.CopyTable @SourceId bigint
 AS 
   SET NOCOUNT ON;
 
   IF OBJECT_ID('tempdb..#MyTable') IS NOT NULL DROP TABLE #MyTable;
 
   SELECT mt.* INTO #MyTable FROM MyTable mt WHERE mt.Id = @SourceId;
 
   ALTER TABLE #MyTable DROP COLUMN Id;
 
   INSERT INTO MyTable SELECT mt.* FROM #MyTable mt;
 
   IF OBJECT_ID('tempdb..#MyTable') IS NOT NULL DROP TABLE #MyTable;
 
   SET NOCOUNT OFF;
 GO
          </p>
         </div>
         <p>
          Ein weiterer Vorteil ist das einfach Aktualisieren der neuen Datenmenge in der temporären Tabelle.
         </p>

          </table>
          <p class="theCode" style="display:none;">
           var enumTest = Big.Small;
 Console.WriteLine(enumTest);
 enumTest = Big.Medium;
 Console.WriteLine(enumTest);
 enumTest = Big.Large;
 Console.WriteLine(enumTest);
          </p>
         </div>
         <p>
          zeigt aber dann das Problem. Die Definition Medium und Large sind intern identisch:
         </p>
         <pre>Small
 Medium
 Medium</pre>
         <p>
          Zur Lösung des Problems muss man zwei Änderungen durchführen. Einmal ist der Aufzählungstyp von long abzuleiten und die Basiszahl für die Bitverschiebung muss ebenfalls eine
          <a href="http://msdn.microsoft.com/de-de/library/ctetwysk.aspx">
           long-Zahl
          </a>
          sein:
         </p>
         <div class="wp_syntax" style="position:relative;">
          <table>
           <tr>
            <td class="code">
             <pre class="csharp" style="font-family:monospace;"><span style="color: #0600FF; font-weight: bold;">public</span> <span style="color: #6666cc; font-weight: bold;">enum</span> Big<span style="color: #008000;">:</span> <span style="color: #6666cc; font-weight: bold;">long</span>
 <span style="color: #008000;">{</span>
     Small <span style="color: #008000;">=</span> 1L <span style="color: #008000;">&lt;&lt;</span> <span style="color: #FF0000;">1</span>,
     Medium <span style="color: #008000;">=</span> 1L <span style="color: #008000;">&lt;&lt;</span> <span style="color: #FF0000;">8</span>,
     Large <span style="color: #008000;">=</span> 1L <span style="color: #008000;">&lt;&lt;</span> <span style="color: #FF0000;">40</span>
 <span style="color: #008000;">}</span></pre>
            </td>
           </tr>
          </table>
          <p class="theCode" style="display:none;">
           public enum Big: long
 {
     Small = 1L &lt;&lt; 1,
     Medium = 1L &lt;&lt; 8,
     Large = 1L &lt;&lt; 40
 }
          </p>
         </div>
         <p>
          Das Ergebnis bestätigt die korrekte Ausführung. Bei der Nutzung von 64 Bit ist aber dann vorerst wirklich Schluss!
         </p>
         <div class="fixed">

        <h2>
         <a class="title" href="https://blog.topdf.de/?p=618" rel="bookmark">
          NDepend, ein Einblick
         </a>
        </h2>
        <div class="info">
         <span class="date">
          27. April 2014
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=618#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Als ich vor kurzem gebeten wurde, mir
          <a href="http://www.ndepend.com/">
           NDepend
          </a>
          anzuschauen, sagte ich sofort zu. Diese Software versprach mir Unterstützung in der Entwicklung und Überwachung von sauberem Code, ganz im Sinne von
          <a href="http://blog.topdf.de/?tag=ccd">
           CCD
          </a>
          . Nach dieser Sichtung habe ich verstanden, dass die Autoren der Software eine großartige Arbeit geleistet haben. Man erhält ein ungeheuer ausgefeiltes Stück Software, das den Entwickler zu Anfang mit der Fülle und dem Umfang von Dialogen und Optionen schlicht überfährt. Man lernt aber schnell, dass es scheinbar unvermeidbar war, damit die Software in unterschiedlichen Bedingungen eingesetzt werden kann. Ich rate jedem, der seinen NET-Sourcecode verbessern möchte oder im Team Fehlentwicklungen im Auge behalten will, sich die Zeit der Einarbeitung zu nehmen. Es lohnt sich!
         </p>
         <p>
          <strong>
           Überprüfung des eigenen Codes
          </strong>
         </p>
         <p>
          Als Einstieg sollte man sich diese Seite anschauen:
          <a href="http://www.ndepend.com/GettingStarted.aspx">
           GettingStarted
          </a>
          . Dort wird als Video und alternativ in einer reich bebilderte Anleitung dargestellt, wie leicht NDepend mit dem eigenen Projektmappen verbunden werden kann. Dieser Vorgang funktioniert sehr gut und schnell. Die Original-VS-Projekte werden nicht angefasst, die NDepend-Verknüpfung kann auch leicht wieder gelöst werden. Eine
          <a href="http://tortoisesvn.net/">
           SVN
          </a>
          -Integration wird dadurch nicht beeinträchtigt.
         </p>
         <p>
          <a href="http://blog.topdf.de/wp-content/uploads/2014/04/NDependBig10.jpg">
           <img alt="NDepend DashBoard" class="alignleft size-medium wp-image-621" height="195" sizes="(max-width: 300px) 100vw, 300px" src="http://blog.topdf.de/wp-content/uploads/2014/04/NDependBig10-300x195.jpg" srcset="https://blog.topdf.de/wp-content/uploads/2014/04/NDependBig10-300x195.jpg 300w, https://blog.topdf.de/wp-content/uploads/2014/04/NDependBig10-1024x666.jpg 1024w, https://blog.topdf.de/wp-content/uploads/2014/04/NDependBig10.jpg 1100w" width="300"/>
          </a>
          Dann kann man sich bereits mittels des über das NDepend-Menü erreichbaren DashBoards einen Überblick verschaffen. Diese Beispielgrafik von NDepend stellt natürlich einen späteren Stand dar, zeigt aber die Fülle Informationen: Für mich war besonders wichtig, dass man sogar einen Überblick erhält, ob im Lauf der Zeit immer mehr Regeln verletzt wurden. Das hilft in der Team-Kontrolle! Weiterhin kann aus diesem DashBoard heraus leicht geprüft werden, welche Regeln verletzt wurden und vor allen Dingen: an welcher Stelle im Source! Regeln, welche man nicht für wichtig erachtet, kann man leicht deaktivieren. Alle Regeln liegen auch in einer gut kommentierten LINQ-ähnlichen Sprache vor, was die Anpassung leicht macht. Hier steht zusätzlich zu jeder NDepend eigenen Funktion eine gute Hilfe im Maus-Tooltip zur Verfügung.
         </p>
         <p>
          <strong>
           Vergleichen der Entwicklungsgeschichte
          </strong>
         </p>
         <p>
          <a href="http://blog.topdf.de/wp-content/uploads/2014/04/NDependHist.jpg">
           <img alt="NDepend Historic Analysis Result" class="alignright size-medium wp-image-626" height="187" sizes="(max-width: 300px) 100vw, 300px" src="http://blog.topdf.de/wp-content/uploads/2014/04/NDependHist-300x187.jpg" srcset="https://blog.topdf.de/wp-content/uploads/2014/04/NDependHist-300x187.jpg 300w, https://blog.topdf.de/wp-content/uploads/2014/04/NDependHist.jpg 642w" width="300"/>
          </a>
          Man stellt schnell fest, dass es nicht immer sinnvoll und praktikabel ist, gleich den kompletten Code Regelkonform zu gestalten, sprich alle Regel-Verletzungen zu entfernen. NDepend bietet hier die Möglichkeit, Basisstände als Vergleich heranzuziehen. Somit werden nur noch neuere Regelverletzungen angezeigt, was natürlich gerade für den Start sinnvoll ist. Als Start kann man dazu, wie in der nebenstehenden Grafik dargestellt, die historische Speicherung der Werte auf
          <em>
           Always
          </em>
          stellen. Damit hat man bereits nach der ersten NDepend-Überprüfung einen Vergleichsstand zur Verfügung, was den Start stark erleichtert.
         </p>
         <p>
          <strong>
           Bewertung
          </strong>
          <br/>
          NDepend eignet sich nicht für den Programmiereinsteiger. Man muss ein Interesse an Clean Code haben und auch schon diverse Regeln kennen und anwenden, LINQ darf kein Fremdwort mehr sein. Dann bietet NDepend eine sehr gute Untersützung. Wenn man für diesen Schritt bereit ist, bietet das Tool einen hervorragenden Start in die permanenten Prüfung und Überwachung von Code-Regeln, zumal sogar für die Integration in Buildserver gesorgt wurde. NDepend liefert ein umfangreiches Set an Regeln, sauber gruppiert, bereits mit. Eigene Regeln können aus dem Code extrahiert werden. Als absolutes Power-Feature bewerte ich die Definition einer Baseline für vergleichende Berichte. Damit können alte Probleme leichter ausgeblendet werden.
         </p>
         <p>
          Eine Bewertung zum Preis fällt schwer: Die Dialog von NDepend wurden mit sehr viel Liebe erstellt. Praktisch überall stehen sinnvolle Optionen, Verknüpfungen zu Hilfethemen und Tooltips zur Verfügung. Der Preis erscheint hierfür eigentlich zu gering. Aus der Sicht der Programmierer, die bei Clean-Code oft nicht gerade offene Türen einrennen, ist der Preis dann wieder wirklich sinnvoll und mit Blick auf den Funktionsumfang mehr als fair.
         </p>

        <div class="info">
         <span class="date">

         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">

        <div class="content">
         <p>
          Ein neuer Windows 2012 R2 Webserver zeigte per Default fleißig alle angelegten User an. Um das Problem in den Griff zu bekommen, genügt ein Aufruf der
         </p>
         <pre>secpol.msc</pre>
         <p>
          Dort kann man dieser
          <a href="http://social.technet.microsoft.com/Forums/windowsserver/en-US/55bf8e73-1e78-45f1-afe2-803a8db6ad90/classic-logon">
           Anleitung
          </a>
          folgen, entnommen von
          <a href="http://social.technet.microsoft.com/">
           http://social.technet.microsoft.com/
          </a>
         </p>
         <blockquote>
          <p>
           Open Computer Configuration, Security Options and on the right hand side, scroll down to Interactive logon: Do not display last user name. Double click on this entry and you will be presented with a dialog box that has two options – Enabled and Disabled (default). Change this setting to Enabled, and then click on the OK button.
           <br/>
           Open Interactive logon: Do not require CTRL+ALT+DEL and select the Disable option and click OK.
          </p>
         </blockquote>

        <h2>
         <a class="title" href="https://blog.topdf.de/?p=599" rel="bookmark">
          SQL User mit Login verbinden
         </a>
        </h2>
        <div class="info">
         <span class="date">
          12. Februar 2014
         </span>
         <span class="author">
          <a href="https://blog.topdf.de/?author=1" rel="author" title="Beiträge von Mario Noack">
           Mario Noack
          </a>
         </span>
         <span class="comments">
          <a href="https://blog.topdf.de/?p=599#respond">
           Keine Kommentare
          </a>
         </span>
         <div class="fixed">
         </div>
        </div>
        <div class="content">
         <p>
          Regelmäßig nach einer Wiederherstellung einer bestehenden Datenbank auf einem neuen Server müssen die Logins, die serverseitig gespeichert und angelegt werden, mit dem User der Datenbank verbunden werden. Ansonsten erhält der Login auf die Datenbanken einfach keinen Zugriff.
         </p>
         <p>
          Dafür gibt es drei Möglichkeiten:
         </p>
         <ul>
          <li>
           Man kann den User in der Datenbank entfernen und den User einfach auf Serverebene
           <a href="http://www.binary.ag/index.php?id=202" target="_blank">
            neu anlegen
           </a>
           . Das SQL Managmentstudio bietet hierbei die Möglichkeit, die Rechte auf einer Datenbank zu erteilen. Aber dabei verliert man ALLE bereits konfigurierten Rechte, die der User in der Datenbank hatte. Diese Lösung ist somit nicht empfehlenswert.
          </li>
          <li>
           Der Microsoft SQL Server bietet die Procedure
           <a href="http://technet.microsoft.com/de-de/library/ms174378.aspx" target="_blank" title="sp_change_users_login (Transact-SQL)">
            sp_change_users_login
           </a>
           , mit der ein neu angelegter Login im Server mit dem User der Datenbank verbunden werden kann. Ich nehme dafür eigentlich immer einen identischen Namen, um Verwechslungen zu vermeiden:
           <div class="wp_syntax" style="position:relative;">
            <table>
             <tr>
              <td class="code">
               <pre class="sql" style="font-family:monospace;"><span style="color: #993333; font-weight: bold;">EXEC</span> sp_change_users_login <span style="color: #ff0000;">'Update_One'</span><span style="color: #66cc66;">,</span> <span style="color: #ff0000;">'User1'</span><span style="color: #66cc66;">,</span> <span style="color: #ff0000;">'User1'</span>;</pre>
              </td>
             </tr>
            </table>
            <p class="theCode" style="display:none;">
             EXEC sp_change_users_login 'Update_One', 'User1', 'User1';
            </p>
           </div>
           <p>
            Dieser Befehl muss für jeden User in jeder Datenbank ausgeführt werden, kann aber leicht in einem Skript vorbereitet werden.
           </p>
          </li>
          <li>
           Abgelöst wird diese vorstehende Procedure durch folgenden SQL Befehl:
           <a href="http://technet.microsoft.com/de-de/library/ms176060.aspx" target="_blank">
            ALTER USER
           </a>
           . In der Anwendung kann man dann schreiben:
           <div class="wp_syntax" style="position:relative;">
            <table>
             <tr>
              <td class="code">
               <pre class="sql" style="font-family:monospace;"><span style="color: #993333; font-weight: bold;">ALTER</span> <span style="color: #993333; font-weight: bold;">USER</span> User1 <span style="color: #993333; font-weight: bold;">WITH</span> LOGIN <span style="color: #66cc66;">=</span> User1;</pre>
              </td>
             </tr>
            </table>
            <p class="theCode" style="display:none;">
             ALTER USER User1 WITH LOGIN = User1;
            </p>
           </div>
           <p>
            Auch dieser Befehl muss für jeden User in jeder Datenbank ausgeführt werden und kann leicht in einem Skript vorbereitet werden.
           </p>
          </li>
         </ul>

         </div>
